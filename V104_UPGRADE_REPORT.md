## 🏗️ 核心模块升级

### 1️⃣ 凯利仓位管理系统

**核心公式**：
```
f* = (bp - q) / b
```

其中：
- f* = 最优仓位比例
- b = 赔率（盈亏比）
- p = 胜率
- q = 1 - p（败率）

**实践参数**：
- 凯利系数：0.5（1/2凯利保守策略）
- 单笔最大亏损：2.5%
- 日亏最大限制：5%
- 最大持仓：25%

**测试结果（来自诊断报告数据）**：

| 品种+周期 | 胜率 | 盈亏比 | 全凯利仓位 | 1/2凯利仓位 |
|---------|------|--------|----------|-----------|
| SOLUSDT 1H | 44.4% | 2.26 | 44.7% | **22.3%** |
| BNBUSDT 1H | 54.5% | 1.41 | 31.3% | **15.6%** |
| BTCUSDT 5m | 52.9% | 1.17 | 14.8% | **7.4%** |
| BTCUSDT 1H | 42.9% | 1.53 | 8.5% | **4.2%** |

**学术依据**：
> "满仓玩家93%的账户经历过清算，而仓位≤15%的交易者在牛市中捕获率达到78%。" - 加密货币交易风险数据

#### 三大核心交易模型

**① 增量模型（Incremental Model）**

| 特征 | 描述 |
|------|------|
| 风险收益比 | 低，每笔捕捉小额可预测收益 |
| 胜率 | 高（约65%+） |
| 频率 | 中频 |

核心信号：
- RSI超买超卖（14周期）
- 均值回归（Z-score）
- 订单簿失衡分析

**② 凸性模型（Convex Model）**

| 特征 | 描述 |
|------|------|
| 风险收益比 | 高盈亏比（5:1+） |
| 胜率 | 中等 |
| 频率 | 低频 |

核心信号：
- 布林带突破（20周期，2标准差）
- ADX趋势强度（阈值25）
- 突破确认

**③ 专业化模型（Specialist Model）**

| 特征 | 描述 |
|------|------|
| 风险收益比 | 危机Alpha获取 |
| 胜率 | 极端事件 |
| 频率 | 超低频 |

核心信号：

**学术依据**：

**核心参数**：
- 网格数量：15-30层
- 网格间距：波幅的0.5%-2%
- 单网格资金：账户余额的1%-2%
- 网格范围：基于ATR动态调整

**示例（ETH波幅200美元）**：
- 网格间距：10-20美元
- 保证触发频率
- 避免手续费过高

### 4️⃣ 统计套利模块

**适用场景**：
- ETH各周期全面亏损
- 放弃单边方向策略
- 改用跨品种价差交易

**核心逻辑**：
```
价差 = 资产1价格 - 资产2价格
Z-score = (价差 - 均值) / 标准差
```

**信号生成**：
- Z-score > 2.0 → 做多资产2，做空资产1
- Z-score < -2.0 → 做多资产1，做空资产2
- \|Z-score\| < 0.5 → 平仓

1. **凯利公式优化**
   > "全凯利仓位在加密货币市场中风险较高，专业交易者普遍推荐1/2凯利或1/4凯利保守策略。"

2. **动态网格交易（2025）**
   > "传统静态网格交易在简单假设下期望收益接近零。学术界在2025年提出了动态网格交易策略（DGT），通过对网格位置的动态重置形成趋势跟踪效果。"

3. **自适应双均线优化**
   > "2025年的一项学术研究表明，经典双均线策略（2-SMA）在结合基于学习的元启发式优化器后，在未见测试期间的平均投资收益率达到7.9%，最高达17.2%。"

4. **样本量统计学**
   > "要做出具备统计置信度的结论，通常需要200~500笔以上的回测样本。"

5. **1分钟周期研究**
   > "1分钟时间框架是高频交易和算法机器人的专属区域，不应供普通手工交易使用。超高频机器人中仅约40%实现盈利，平均收益率约为-10%。"

## ⚠️ 注意事项

### 使用建议

1. **凯利仓位管理**
   - 必须使用1/2或1/4凯利保守策略
   - 单笔最大亏损不超过2.5%
   - 日亏不超过5%

2. **差异化策略**
   - 不同品种使用不同策略
   - 不要"单一策略打天下"
   - 策略多样化和风险贡献均匀分配

3. **样本量验证**
   - 回测必须达到200-500笔
   - 严格执行样本内外分割
   - 防止过拟合

4. **ETH交易**
   - 放弃单边方向策略
   - 使用跨品种统计套利
   - 与BTC/SOL组合
