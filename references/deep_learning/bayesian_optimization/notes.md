## 一、概念理解

### 核心问题

交易系统有大量超参数（RSI周期、ATR倍数、BB标准差、仓位上限等）。网格搜索在3-4个参数时还行，超过8个参数就指数爆炸——100^8 = 10^16 次评估。贝叶斯优化的核心思想是：**用更少的评估找到更好的参数**。

### 为什么比网格/随机搜索更聪明

网格搜索：均匀采样，不利用每次评估的结果
随机搜索：随机采样，略好但不学历史
贝叶斯优化：**利用每次评估结果指导下一次采样**

关键洞察：目标函数 f(x) 是 expensive-to-evaluate（每次回测可能需要几小时），所以要最小化评估次数。

### 两层框架

```
观察: f(x) 在 5 个点上的值
    ↓
1. 代理模型(Surrogate): 建立 f(x) 的概率模型
   常用: 高斯过程(GP) → 输出均值 μ(x) 和不确定性 σ(x)
   
    ↓
2. 采集函数(Acquisition): 决定下一个评估点
   核心: 最大化 acquisition(x) → 选择"最有潜力"的 x
   
    ↓
重复直到收敛或达到评估预算
```

### 采集函数的三种选择

| 采集函数 | 公式 | 特点 |
|----------|------|------|
| **Expected Improvement (EI)** | E[max(f(x) - f_best, 0)] | 最常用，自然平衡探索利用 |
| **Upper Confidence Bound (UCB)** | μ(x) + κ·σ(x) | κ大→探索，κ小→利用 |
| **Probability of Improvement** | P(f(x) > f_best) | 保守，只关心超越当前最优 |

**EI 是交易参数优化的首选**，因为它同时考虑了改进幅度和概率。

### 与系统的关系

当前系统 `optimizer_bayes.py` 使用 bayesian-optimization 库（基于 GP+EI），支持：
- 参数搜索空间定义（pbounds）
- 30次迭代 + 5个初始随机点
- 自动返回最优参数

系统还**缺少**：
- HRP聚类阈值优化（linkage方法、单/complete/average）
- Hawkes衰减参数优化（μ, α, β 三个参数）
- 策略切换阈值优化（市场状态边界）
- 目标函数自定义（支持夏普/胜率/最大回撤组合）

## 二、论文精读

### 论文1: 基础框架 — "Practical Bayesian Optimization" (Snoek et al., 2012)

奠基论文。核心贡献：
- 证明GP+EI在超参数优化上优于随机搜索和网格搜索
- 首次将BO用于机器学习超参数（MCMC、CNN）
- 关键技术：MCMC采样处理超参数的不确定性

关键公式（Expected Improvement）：

```
EI(x) = E[max(f(x) - f_best, 0)]
      = (μ(x) - f_best - ξ) · Φ(Z) + σ(x) · φ(Z)   (当 σ > 0)
      
其中 Z = (μ(x) - f_best - ξ) / σ(x)
      ξ = exploration parameter (默认0.01)
      Φ = 标准正态CDF
      φ = 标准正态PDF
```

**脆弱性**：GP的协方差函数选择影响很大，RBF核在周期性强参数上表现差。

### 论文2: 交易应用 — "Bayesian Optimization in Trading" (CSAI/Various, 2021+)

交易领域的BO应用总结：
- 目标函数通常是多目标的（收益+风险+频率）
- 约束条件多（风控硬参数不可调）
- 评估噪声大（市场非平稳）

关键发现：
- BO在5-10维参数空间效果最好
- 超过20维时，探索-利用平衡变困难
- 建议将参数分组，每组独立BO

### 论文3: 过拟合检测 — "A Comparison of EMCO and Random Search" (约2019)

对比了不同采集函数在交易中的应用：
- EI 倾向于 exploitation（利用），在多峰函数上可能陷入局部最优
- UCB 的 κ 参数需要手动调
- **建议：先用EI探索，再用UCB精细化**

## 三、知识关联

### 与 Cycle 1 (HRP) 的连接

HRP有两个关键超参数：
1. **聚类方法**：`linkage` 方法（single/complete/average/ward）——离散选择，不适合BO
2. **距离矩阵**：`correlation` vs `euclidean` ——离散选择

**更值得BO的HRP相关参数**：
- HRP输出后应用的最大/最小权重约束（max_weight_ratio）
- 协方差矩阵的 ewma_lambda（0.01-0.2）
- 风险平价目标（等风险 vs 等波动）

### 与 Cycle 2 (Hawkes) 的连接

Hawkes过程有三个连续参数，是BO的理想目标：

```
λ(t) = μ + α · ∫₀ᵗ β·e^(-β(t-s)) dN(s)

μ (基强度): 0.001-0.1       → 控制背景事件率
α (兴奋系数): 0.0-0.99      → 控制自激强度  
β (衰减率): 0.1-10.0        → 控制衰减速度
```

这三个参数的组合空间是连续的，且真实评估（用最大似然估计）很慢——完美适配BO场景。

## 四、独立复述

### 我真正理解了什么

贝叶斯优化的本质是**"学会学习"**——不是每次采样都重新开始，而是利用所有历史数据构建对目标函数的认知，然后主动选择最有价值的下一个采样点。

**GP的价值**：在每个点返回 μ（预测均值）和 σ（预测标准差）。σ高=不确定性大=值得探索；μ高=预测好=值得利用。采集函数是这两者的平衡。

**EI的直觉**：与其只关心"哪个点最好"，不如关心"哪个点最有可能改进"。改进幅度越大、概率越高，EI越大。这自然地在探索（改进潜力大）和利用（接近已知最优）之间平衡。

### 之前误解了什么

**误解1**：BO可以直接用于任何黑盒函数
**真相**：BO假设函数相对平滑。完全随机、高频振荡的函数（如加密货币分钟级数据）会让GP拟合很差。需要先对数据降采样或对目标函数做平滑。

**误解2**：BO一定能找到全局最优
**真相**：GP+EI仍然是局部搜索的近似。在高度多峰的非凸函数上，可能陷入局部最优。建议多次重启BO（用不同随机种子）。

### 真实市场的脆弱性

**非平稳性是最大威胁**：今天找到的最优参数，下周可能因为市场结构变化而失效。这不是BO本身的问题，而是回测-实盘差异的系统性问题。

**解决方案**：
- 滚动窗口BO：只用最近N个月的数据
- 贝叶斯模型平均：保留多组参数，按近期表现加权
- Safe BO：在安全边界内搜索，不越界

## 六、关键公式汇总

### 高斯过程（GP）预测
```
f(x*) ~ N(μ(x*), σ²(x*))

μ(x*) = k(x*, X) · K(X,X)⁻¹ · y
σ²(x*) = k(x*, x*) - k(x*, X) · K(X,X)⁻¹ · k(X, x*)
```

其中 k(x, x') 是核函数（常用 RBF/马特恩）

### Expected Improvement
```
EI(x) = max(σ(x) · Z · Φ(Z) + (μ(x) - f_best - ξ) · Φ(Z), 0)
Z = (μ(x) - f_best - ξ) / σ(x)
```

### UCB
```
UCB(x) = μ(x) + κ · σ(x)
```
