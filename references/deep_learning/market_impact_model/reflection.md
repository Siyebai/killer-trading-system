# 市场冲击模型深度学习反思

## 一、我真正理解了什么

### 1.1 冲击模型的本质是"对供需失衡的定价"

冲击不是简单的"交易成本"，而是对市场流动性的定价。当我的订单量相对于市场日均成交量（ADV）很大时，我的订单本身改变了供需平衡，价格会沿着不利于我的方向移动。这是一种**内生性成本**——我自己制造了我要承受的成本。

SquareRoot模型的直觉是：冲击与`√(Q/ADV)`成正比。这意味着：
- 小单（Q/ADV=0.1%）的冲击与`√0.001`成正比 → 相对温和
- 大单（Q/ADV=10%）的冲击与`√0.1`成正比 → 显著放大
- 最关键的是：冲击不是线性增长的，是**亚线性**的（平方根而非线性）

这就是为什么拆分订单有效——把1个大单拆成10个小单，冲击从`√10`变成`10×√0.1`，改善了约3倍。

### 1.2 AC模型的核心价值是框架而非精确预测

Almgren-Chriss模型给了我一个关键的洞见：**执行时间的选择本身就是风险**。在金融市场中，时间=不确定性=金钱。所以最优执行不只是"如何下单"，而是"在多长时间内下多少单"。

具体来说：
- **快执行**：冲击小（因为价格不会等太久就移动了），但方差大（因为价格在这段时间内可能大幅波动）
- **慢执行**：冲击大（因为持续买入推高了价格），但方差小（因为摊薄了波动）

AC模型用`λ`（风险厌恶参数）在"冲击-方差权衡"之间找到最优路径。这是一个**框架思维**，不是"查表得出最优答案"。

### 1.3 实证发现与理论直觉的差异

我之前直觉认为AC模型应该给出比SquareRoot更高的冲击（因为AC考虑了"慢执行=冲击大"）。但实验结果显示，在当前的参数配置下，AC给出了**最低**的冲击估算。

这让我重新理解了模型之间的关系：
- SquareRoot是一个**现象学模型**——直接拟合"订单规模→价格偏移"的实证规律
- AC是一个**机制模型**——从微观结构推导"投资者最优行为"的均衡结果

两者描述的不是同一个东西。SquareRoot描述的是"我下这个单，对市场的影响是什么"；AC描述的是"如果我要最大化效用，应该如何执行"。机制模型需要参数校准才能转换为现象学预测。

## 二、我之前对什么有误解

### 误解1："冲击成本是固定的"

我之前以为"冲击成本=滑点"是一个固定的百分比（如0.05%）。实际上，冲击是**订单规模的函数**——同一标的，1 BTC vs 0.001 BTC的冲击可能相差100倍。这意味着：
- 小单不需要担心冲击（Linear模型足够）
- 大单必须用SquareRoot模型估算冲击
- 系统中**不能用统一滑点**处理所有订单

### 误解2："AC模型比SquareRoot更精确"

我以为AC模型是"更高级的"SquareRoot，会给出更准确的冲击预测。实际上，两者描述的是不同的东西。AC的真正价值在于提供**执行策略优化**（何时下单、下多少），而不是冲击的绝对数值预测。

### 误解3："市场冲击对所有订单都一样"

冲击成本在不同市场状态下差异巨大（实验显示可达134倍）。固定滑点模型完全无法捕捉这一点。系统需要能**感知市场状态**的冲击模型。

## 三、理论在真实市场中的最大脆弱性

### 脆弱性1：相关性假设失效

所有冲击模型都隐含假设"波动率和成交量是独立的"。但真实市场存在**波动率-成交量联动**（V集），即高波动期往往伴随低流动性（特别是在加密货币市场）。这意味着最需要冲击保护的时候（高波动），恰好也是冲击最严重的场景。

SquareRoot模型用`σ×√(Q/ADV)`分别处理波动率和流动性，但没有捕捉到两者之间的**非线性耦合**。在高波动+低流动性的极端场景下，实际冲击可能是模型预测的2-3倍。

### 脆弱性2：顺序执行假设

SquareRoot和AC模型都假设订单是"单独执行"的。但在真实市场中，当多个参与者同时有大单需要执行（常见于期货到期日、期权到期日附近），市场深度会被**多重占用**，导致冲击远超模型预测。

### 脆弱性3：信息泄露

模型假设执行是"隐蔽的"。但大单的拆单执行会在订单簿上留下痕迹（持续买入行为会被识别），导致其他交易者"搭便车"或"反向操作"，额外增加冲击。这在高频环境中尤为明显。

## 四、实验中理论与实践的差距

### 差距1：参数校准问题

实验中使用的参数（`σ=0.02`, `γ=0.1`等）是假设值。真实部署需要：
- 用历史TCA数据拟合`σ`（波动率参数）
- 用历史数据拟合`η`（临时冲击系数）
- 用历史数据拟合`γ`（永久冲击系数）

这是一个**监督学习**问题，不是理论推导问题。

### 差距2：单笔vs多笔叠加效应

实验模拟的是单笔订单的冲击。在真实交易中，同一策略可能连续发出多个信号，导致：
- 连续买入推高成本
- 策略间的冲击叠加
- 账户层面的暴露增加

### 差距3：做空机制的特殊性

数字货币的做空机制（借币卖出）引入了额外的复杂性：
- 借币成本（利率）
- 流动性不对称（做空时买入平仓在流动性差时更困难）
- 强制平仓风险在暴涨时

## 五、整合到系统的关键决策

### 决策1：SquareRoot作为主模型

**理由**：实证支持最强，计算简单，对大单敏感，与现有EV过滤框架集成最自然。

**风险**：参数未校准，初期可能系统性偏差 → 通过TCA日志逐步校准。

### 决策2：先加建议层，再决定自动化程度

**理由**：冲击模型在真实数据上的表现尚未验证，过早全自动化可能导致意外拒绝好信号。

**路径**：
1. Phase 1：仅在日志中记录预测冲击，不影响决策
2. Phase 2：作为建议展示，供人工参考
3. Phase 3：根据TCA校准结果，对明显错误预测做兜底过滤
4. Phase 4：完全自动化（仅在积累足够TCA数据后）

### 决策3：TCA日志是必要基础设施

**理由**：没有TCA反馈，无法知道模型预测的准确性，无法校准参数，无法识别模型失效的市场状态。

**实现优先级**：高（应该作为Phase 1的第一步）

## 六、下一步学习建议

### 下一个主题：过拟合检测方法比较

风险平价学到了"如何分配资金"，市场冲击学到了"如何执行订单"。但还有一个核心问题没有解决：**我怎么知道我的策略是否过拟合了？**

这是一个检验"策略质量"而非"策略设计"的问题。当前系统的策略实验室（strategy_lab.py）还没有系统的过拟合检测机制。

推荐学习主题：**CSCV（组合对称交叉验证）和PBO（概率化击败比例）**

核心问题：
- 给定我的策略在历史数据上表现X，它是真实的Alpha还是过拟合的结果？
- 我的策略相比基准（买入持有、1/N等）真正有优势的概率是多少？
- 策略衰减的速度如何预测？

这个主题与当前系统直接相关：
- 可以直接集成到strategy_lab.py中
- 可以对所有历史策略做"过拟合概率"评分
- 可以为策略退役决策提供量化依据
